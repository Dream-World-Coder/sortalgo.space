## Key Decisions to Discuss:

### 1. **Content Management**
- **Option A:** Pure Markdown files in `/content/chapters/` (simpler, git-friendly)
- **Option B:** MDX (Markdown + JSX components for interactive demos)
- **My recommendation:** Start with pure Markdown + KaTeX, add MDX later if you need interactive visualizations

### 2. **Chapter Metadata**
Should we use:
- **Frontmatter** in each .md file (title, date, order, description)?
- **Separate metadata.json** for chapter ordering?
- **Both?** (frontmatter for content-specific data, separate file for ordering)

Example frontmatter:
```yaml
---
title: "Bubble Sort: Understanding the Basics"
chapter: 2
difficulty: "beginner"
date: "2024-01-15"
description: "A deep dive into bubble sort with time complexity analysis"
---
```

### 3. **Navigation Structure**
- Linear (Previous/Next buttons only)?
- Sidebar with expandable sections (Basics â†’ Intermediate â†’ Advanced)?
- Both?

### 4. **Math Rendering**
- Use `remark-math` + `rehype-katex` for server-side rendering?
- Or client-side KaTeX rendering?
- Server-side is better for performance

### 5. **Collaboration Features**
- Would you want comments/discussions per chapter eventually?
- GitHub edit links for each page?
- Just static content for now?

## My Recommendations:

1. **Content in `/content/chapters/`** - Easy to edit, version control friendly
2. **Frontmatter for metadata** - Keeps everything in one place
3. **Server-side markdown rendering** - Better performance, SEO
4. **Sidebar + linear navigation** - Easy chapter browsing
5. **Group chapters by difficulty** in metadata
